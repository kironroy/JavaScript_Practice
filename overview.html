Basics

  • JavaScript is a high-level programming language. It abstracts memory management through garbage collection. Automatic reclaiming of unused objects.
  • It is prototype-based. It uses an object-oriented approach. Almost everything is an object. The exceptions are primitive values: numbers and strings.
  • Blueprints create arrays and objects. Arrays originate from an array blueprint.
  • JavaScript supports first-class functions. Functions can pass as arguments or returned by other functions for enhanced flexibility.
  • JavaScript is dynamically typed. No need to explicit type definitions for variables.
  • Its concurrency model defines how the engine manages many tasks.
  • JavaScript executes in a single thread. It uses an event loop to handle time-consuming operations in the background. Then it reintegrates them into the main thread upon completion.
  • The V8 engine, executes JavaScript code.
  • Modern JavaScript engines use just-in-time (JIT) compilation. The entire code base converted into machine code. This happens fast.

Memory management

Primitives

Primitives are in the Call Stack (References to Objects)

  • Number
  • String
  • Boolean
  • Undefined
  • Null
  • Symbol
  • Big Int

Objects

Objects are in the heap

  • Object literals
  • Arrays
  • Functions

For different type of values ➡️ Memory is allocated in different places.

  • It abstracts memory management through garbage collection. Automatic reclaiming of unused objects.
  • Developers can influence garbage collection by removing references to objects. This allows the engine to reclaim memory.
  • Memory leaks can occur if references to unused objects are not removed. Garbage collection cannot reclaim memory for these objects.
  • Tools like Chrome DevTools can help identify and resolve memory leaks.

Memory life cycle

Memory allocated when creating a variable. It remains allocated until the variable goes out. Whenever a developer creates a variable, the JavaScript engine allocates memory for it.

let x = 10; allocates memory for the variable x.

Memory use

While the code runs, it writes, reads, and updates values in the designated memory space.

x = x + 6; round(x);

Release of Memory

When the variable is no longer needed the value gets deleted from memory to free up resources. The released memory used for new values.

Call Stack: variable environment is simply deleted when Execution Context pops a variable off the stack

Garbage Collection: a central memory management tool in every JavaScript engine. It reclaims unused heap memory without developer control. Factors:

  • How much memory as app is using.
  • Which engine is being used.

All model JavaScript engine use a mark-and-swap algorithm.

  1. Mark: all objects that are reachable from a root as "alive."
  2. Sweep: delete un-marked (unreachable) objects and reclaim memory allocations.

Memory Leaks

Memory leaks occur when objects are no longer needed by the application. Garbage collection tries to reach an object because the object marked as alive. This adds to clutter.
