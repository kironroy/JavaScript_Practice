Basics

  • JavaScript is a high-level programming language that abstracts memory management through garbage collection, automatically reclaiming unused objects.
  • It uses a prototype-based, object-oriented approach where nearly everything is treated as an object, except for primitive values like numbers and strings.
  • Arrays and objects are created from blueprints; arrays originate from an array blueprint.
  • JavaScript supports first-class functions, allowing functions to be passed as arguments or returned by other functions for enhanced flexibility.
  • As a dynamically typed language, JavaScript does not require explicit type definitions for variables.
  • Its concurrency model defines how the engine manages multiple tasks.
  • JavaScript executes in a single thread and uses an event loop to handle time-consuming operations in the background, reintegrating them into the main thread upon completion.
  • The JavaScript engine, such as the widely used V8 engine, executes JavaScript code.
  • Modern JavaScript engines feature just-in-time compilation, converting the entire codebase into machine code instantaneously for execution.

Memory management

Primitives

Primitives are in the Call Stack (References to Objects)

  • Number
  • String
  • Boolean
  • Undefined
  • Null
  • Symbol
  • Big Int

Objects

Objects are in the heap

  • Object literals
  • Arrays
  • Functions

For different type of values memory is allocated in different places.

  • JavaScript manages memory automatically through garbage collection, which reclaims memory from objects that are no longer in use.
  • Developers can influence garbage collection by removing references to objects, allowing the engine to reclaim memory.
  • Memory leaks can occur if references to unused objects are not removed, preventing garbage collection.
  • Tools like Chrome DevTools can help identify and resolve memory leaks.

Memory life cycle

A piece of memory is allocated when a variable is created, and it remains allocated until the variable goes out. Whenever a developer creates a variable, the JavaScript engine allocates memory for it.

let x = 10; allocates memory for the variable x.

Memory use

While code is running, the value is written, read and updated in the allocated piece of memory

x = x + 6; round(x);

Release of Memory

When the variable is no longer needed the value is deleted from memory to free up resources. The released memory is used for new values

Call Stack: variable environment is simply deleted when Execution Context pops a variable off the stack

Garbage Collectionis central memory management tool in any JavaScript engine. It acts automatically. Developers cannot control when heap memory is cleaned by Garbage collection
